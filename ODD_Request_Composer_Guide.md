# ODD Request Composer 사용 가이드

> **ODD Request Composer**는 데이터 요구사항을 ODD 기반의 구조화된 형식으로 작성할 수 있는 요청서 작성 도구입니다.
> "미국 야간 보행자 4만 프레임"처럼 자유롭게 쓰던 요청을 조건 선택만으로 체계적으로 정의할 수 있습니다.

## 1. 이런 분께 추천합니다

- ODD가 뭔지 잘 몰라도 **"어떤 조건의 데이터가 필요한지"**는 알고 있는 분
- 데이터 취득 또는 라벨링 요청을 운영팀에 전달해야 하는 알고리즘 담당자
- 여러 조건 조합의 데이터를 한 번에 체계적으로 요청하고 싶은 분

---

## 2. 핵심 개념 3가지

### 시나리오 그룹
하나의 요청서 안에서 **"성격이 다른 조건 묶음"**을 따로 구분한 단위입니다.

예) 같은 요청이라도 "야간 주차장"과 "우천 고속도로"는 조건이 달라서 별도 그룹으로 등록합니다.

### Core ODD
자율주행 운행 설계 영역(ODD) 기준으로 정의된 표준 조건 항목입니다.
**시간대, 날씨, 도로 유형, 조도** 같은 항목들이 있으며 목록에서 선택만 하면 됩니다.

### Extension
Core ODD로 표현할 수 없는 **추가 조건을 자유롭게 정의**할 수 있습니다.
예) 자차 속도, 타겟 물체 종류, 카메라 blur 수준 등

> **조건조합(Combination):** 선택한 조건들의 모든 경우의 수가 자동으로 계산되어 행으로 생성됩니다.
> 예) 날씨(맑음/비) × 시간대(주간/야간) = 4가지 조건조합

## 3. 전체 사용 흐름

```
① 기본 정보 입력
        ↓
② 시나리오 그룹 추가  ← 조건이 다르면 반복
   Region 선택
   → Core ODD 선택
   → Extension 입력 (선택)
   → Scenario 생성
   → 그룹 저장
        ↓
③ 최종 조건조합 테이블에서 수량·우선순위 입력
        ↓
④ Submit → JSON + Excel 다운로드
```

## 4. 단계별 사용법

### ① 기본 정보 입력

| 항목 | 설명 |
|------|------|
| 제목 | 이 요청을 구분할 수 있는 이름 |
| 요청인 이름·이메일 | 담당자 정보 |
| Product | FV(전방 카메라) / SVC(주변 카메라) / Other |
| Scenario | Driving / Parking / Other — 선택하면 관련 ODD 항목이 상단에 표시됩니다 |
| 요청 범위 | 데이터 취득 / 데이터 라벨링 (중복 선택 가능) |
| Feature | 라벨링 요청 시 Feature 이름 추가 (예: 3DP OD, LPSD) |
| 희망 완료일 | 납기 기준일 |

---

### ② 시나리오 그룹 추가

하나의 그룹 = **Region + Core ODD + Extension + 시나리오 생성 + 저장** 순서로 진행합니다.

#### Region 선택 (필수)
미국 / 한국 / 중국 / 일본 / 유럽 / 지역무관 / 그 외(직접 입력) 중 **하나** 선택합니다.

#### Core ODD 선택
- ⭐ 표시 항목은 권장 필수 조건입니다.
- 각 속성에서 **복수 선택** 가능 → 선택한 값 수만큼 조합이 늘어납니다.
- 목록에 없는 값은 **"기타 (직접 입력)"** 옵션으로 추가할 수 있습니다.

#### Extension 입력 (선택)
Core ODD에 없는 조건을 자유롭게 추가합니다.

1. 카테고리 선택 (environmental / target / vehicle / sensor)
2. 표시명 입력 (예: "자차 속도")
3. 값 목록 입력 (예: "저속, 중속, 고속")
4. **Extension 추가** 클릭

#### Scenario 생성
조건 입력이 끝났으면 **🚀 Scenario 생성** 버튼을 누릅니다.
- 미리보기 테이블에서 생성된 조건조합 확인
- 조합 수가 100개 초과 시 안내, 1,000개 초과 시 경고 표시

#### 그룹 저장
시나리오 이름을 입력하고 **✅ 이 시나리오 저장** 버튼을 누릅니다.
저장 후 새 그룹을 추가하거나 현재 그룹을 계속 수정할 수 있습니다.

---

### ③ 최종 조건조합 테이블

모든 그룹의 조건조합이 하나의 테이블로 통합됩니다.

**수량 단위 선택**

| 단위 | 사용 상황 |
|------|-----------|
| frame | 라벨링 Feature별 프레임 수 입력 |
| hour | 시간 단위로 수량 관리 시 |
| case | 케이스 단위로 수량 관리 시 |

**일괄 설정**
조건조합이 많을 때 모든 행에 동일한 수량·우선순위를 한 번에 적용할 수 있습니다.

> ⚠️ 수정 후 반드시 **💾 수량/우선순위 저장** 버튼을 눌러야 반영됩니다.

---

### ④ Submit

필수 항목이 모두 입력되면 **💾 JSON으로 저장** 버튼이 활성화됩니다.
저장 완료 후 **Excel**과 **JSON** 파일을 각각 다운로드할 수 있습니다.

## 5. 자주 쓰는 팁

**조합 수를 줄이려면**
각 속성에서 선택한 값의 개수를 줄이거나, Extension 값 목록의 항목 수를 줄이세요.
조합 수는 모든 선택 값의 곱입니다.
> 예) 날씨 3가지 × 시간대 2가지 × 도로유형 3가지 = 18가지 조합

**같은 Region인데 조건이 다를 때**
같은 Region이라도 조건의 성격이 다르면 시나리오 그룹을 따로 만드는 것을 권장합니다.
(예: 미국 주간 고속도로 그룹 / 미국 야간 주차장 그룹)

**저장 전에 수정하고 싶을 때**
저장된 그룹 목록에서 **✏️ 수정** 버튼을 누르면 해당 그룹의 조건을 다시 편집할 수 있습니다.

**처음부터 다시 시작하고 싶을 때**
사이드바의 **🔄 새로 시작** 버튼을 누르면 모든 입력이 초기화됩니다.

## 6. 산출물 안내

Submit 후 다운로드하는 Excel 파일은 3개 시트로 구성됩니다.

| 시트 | 내용 |
|------|------|
| **요청 요약** | 요청 기본 정보 전체 (요청인, Product, Scenario, 수량 합계 등) |
| **조건조합 목록** | 핵심 산출물. 전체 조건조합 목록 + 수량·우선순위·메모 |
| **선택 요약** | 그룹별 Core ODD 선택 내용 + Extension 목록 |

JSON 파일은 운영팀 시스템 연동 및 버전 관리 용도로 사용됩니다.

## 7. FAQ

**Q. ODD를 몰라도 쓸 수 있나요?**
네. 목록에서 선택하는 방식이라 ODD 지식 없이도 사용할 수 있습니다.
⭐ 표시 항목 위주로 선택하면 최소한의 구조화된 요청이 완성됩니다.

**Q. Core ODD 목록에 원하는 값이 없어요.**
각 속성의 **"기타 (직접 입력)"** 옵션을 선택하면 값을 직접 입력할 수 있습니다.

**Q. 수량을 시나리오별로 다르게 설정하고 싶어요.**
③ 최종 조건조합 테이블에서 각 행의 수량을 개별 입력하면 됩니다.
일괄 설정 후 특정 행만 수정하는 것도 가능합니다.

**Q. 저장한 JSON 파일은 어디에 있나요?**
앱 서버의 `output/` 폴더에 `request_<제목>_<날짜>.json` 형식으로 저장됩니다.
다운로드 버튼으로도 즉시 받을 수 있습니다.

**Q. 요청서를 수정해서 다시 제출하고 싶어요.**
현재 버전에서는 이전 요청 불러오기 기능을 지원하지 않습니다.
수정이 필요하면 새로 작성하거나, 저장된 JSON을 참고해 주세요.
